# SQL 소개

```SQL
SELECT publisher, price
FROM book
WHERE price > 10000
```

### SQL 기능에 따른 분류
* DDL: 데이터 정의어
    * 테이블이나 관계의 구조를 생성하는 데 사용
    * CREATE, ALTER, DROP 문 등이 있다
* DML: 데이터 조작어
    * 테이블에 데이터를 검색, 삽입, 수정, 삭제하는 데 사용
    * SELECT, INSERT, DELETE, UPDATE 문 등이 있다.
    * SELECT 문은 특별히 질의어(query) 라고 한다.
* 데이터 제어어(DCL)
    * 데이터의 사용 권한을 관리하는 데 사용
    * GRANT, REVOKE 문 등이 있음

DB는 한번 만들어놓고 잘 수정을 잘 안하기 때문에 사실 DML을 쓰는경우가 90%다?

# SELECT
DROP은 테이블 자체를 없애는거임

SQL은 대소문자 구분 안함, 근데 보통키워드를 대문자 나머지를 소문자자로 씀

```SQL
SELECT
FROM
WHERE
GROUP BY
HAVING
ORDER BY
```
SELECT문의 가장 기본본적인 구조라 할 수 있다.

프로그램과 데이터가 독립립돼어있는 DB의 구조 덕분에 컴파일 할필요도 없없고, 언어이 자유도 또한 높다.
